Council Report — Architecture Selection

Bharatiya Antariksh Hackathon 2026 · AI exoplanet detection from TESS light curves · 2026-06-24

The question

Which of three candidate architectures should we build to win 1st place, given physicist judges who distrust black-box AI, a ~48h build, scarce labels, single-sector data gaps, crowded-field blending, and correlated stellar noise? The pipeline must detect, classify (transit / eclipse / blend / starspot), estimate parameters, report confidence, and explain itself.

Council verdict — forge the APEX-ORACLE hybrid

Horizon-B multimodal transformer as the detection and classification spine → Horizon-C differentiable batman + SBI as the parameter and confidence head → Horizon-A BLS as the fast candidate generator. Bolt the centroid / difference-imaging vetting module across all of it. This is the only configuration that maximizes accuracy, removes the black-box objection, recovers monotransits, and produces a standout demo.

Do first: lock the data + label + injection-recovery harness. Nothing is provable without it.

The three Horizons

Horizon-A · Velocity

Fast front-end

SavGol detrend + BLS + 1-D TCN dual view.

  • Trains in minutes, runs on CPU
  • SavGol can erase shallow transits
  • Blind to monotransits; capped ceiling

Horizon-B · Apex

Detection spine

GP detrend + Transformer + 1-D ViT + GBT, multimodal.

  • Highest accuracy on blends / rare classes
  • Attention overlays = free XAI
  • Overfit risk on small labels in 48h

Horizon-C · Oracle

Parameter head

Differentiable batman in the loss + GP noise + SBI posteriors.

  • White-box by construction
  • Recovers monotransits; exact posteriors
  • Hardest to engineer / stabilize in time
